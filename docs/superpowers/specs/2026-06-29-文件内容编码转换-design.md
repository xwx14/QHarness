## 1. 背景与目标

CLAUDE.md 编码约定：与 AI 交互的数据统一 UTF-8；文件**内容**按文件实际编码读取后再转 UTF-8 返回给 AI。当前 `ReadFileTool` 只做文本模式读取（CRLF→LF）+ 截断，**未做内容编码转换**——GBK 文件读出来是 GBK 字节，喂给模型就是乱码。

本设计补齐这块：读取后检测文件内容编码（UTF-8 / GBK / UTF-16 含 BOM），统一转 UTF-8 返回。

## 2. 设计决策（已与用户对齐）

- **检测策略**：BOM → UTF-8 严格校验 → GBK 回退 → 原样兜底。
- **支持编码**：UTF-8、GBK、UTF-16 LE/BE（BOM）。
- **失败兜底**：GBK 转换失败 / 二进制文件 → **原样字节返回**（不报错、不崩，模型自行判断）。
- **截断时机**：先转 UTF-8，再**按字符边界截断**（不切断多字节字符）。

## 3. 模块划分

新建 `core/tool/ContentCodec.h`（全 inline，无需 `QH_API` 导出），专注**文件内容**检测+转 UTF-8；与 `PathCodec.h`（**路径名**编码）职责分离。底层 UTF-8↔GBK 复用 `pathCodec::gbkToUtf8`，避免重复。

## 4. 接口

```cpp
namespace qh::tool::contentCodec {
// 检测 raw 的编码并转为 UTF-8（剥离 BOM）。
// 顺序：BOM(UTF-8/UTF-16 LE·BE) → UTF-8 严格校验 → GBK → 原样兜底。
std::string toUtf8(const std::string& raw);
}
```

## 5. 检测与转换流程

`toUtf8(raw)`：

1. **BOM 头**（前 2~3 字节）：
   - `EF BB BF` → UTF-8 BOM，剥 BOM 返回 `raw[3..]`。
   - `FF FE` → UTF-16 LE，剥 BOM，`utf16ToUtf8(raw[2..], true)`。
   - `FE FF` → UTF-16 BE，剥 BOM，`utf16ToUtf8(raw[2..], false)`。
2. **无 BOM**：`isValidUtf8(raw)` 合法 → 直接返回 `raw`。
3. **GBK**：`auto s = pathCodec::gbkToUtf8(raw);` 非空 → 返回 `s`。
4. **兜底**：返回 `raw`（原样字节）。

## 6. 辅助函数（ContentCodec 内，跨平台）

- `void appendUtf8(std::string&, unsigned cp)`：码点 → 1~4 字节 UTF-8。
- `bool isValidUtf8(const std::string&)`：UTF-8 严格逐字节校验——leading/continuation 字节数匹配、overlong 拒绝、越界（>U+10FFFF）拒绝、代理码点（U+D800~U+DFFF）拒绝。纯 ASCII 合法。
- `std::string utf16ToUtf8(const std::string& utf16, bool littleEndian)`：手动解码 UTF-16 码点（含 surrogate pair → 补充平面），逐码点 `appendUtf8`。**不依赖 `wchar_t` 宽度**（Linux=32 / Windows=16 行为一致）。孤立代理码点跳过。
- `std::string truncateUtf8Safe(const std::string&, size_t maxBytes)`：截到 maxBytes，从该处向前回退到字符边界（`s[end]` 非 continuation 字节处），保证不切断多字节字符。

## 7. ReadFileTool 改动

`execute` 读取后插一步编码转换，截断改字符边界安全版：

```cpp
std::string content = contentCodec::toUtf8(rawContent);        // 新增：内容编码转换
if (content.size() > kMaxLen)
    content = contentCodec::truncateUtf8Safe(content, kMaxLen) +
              "\n\n...[由于内容过长，已被系统截断]...";
```

其余流程（参数解析、`resolveWorkBase`/`resolveInside`、错误处理）不变。

## 8. 测试（TDD）

- UTF-8 无 BOM（含中文）→ 原样。
- UTF-8 带 BOM（`EF BB BF`+内容）→ 剥 BOM。
- GBK（"测试" = `B2 E2 CA D4`）→ "测试" UTF-8（`E6 B5 8B E8 AF 95`）。
- UTF-16 LE 带 BOM → 正确中文。
- UTF-16 BE 带 BOM → 正确中文。
- 纯 ASCII → 原样。
- 二进制/非法序列 → 原样兜底，不崩。
- `truncateUtf8Safe`：在多字节字符处截断不切断（截到字符边界）。

## 9. 边界与约束

- `isValidUtf8` 必须严格：否则含偶然非法字节的 UTF-8 文件会被误判为 GBK。
- UTF-16 surrogate pair 须成对；孤立代理视为无效（不输出对应字符）。
- 兜底原样保证二进制文件（图片等被误读时）不崩，但返回内容可能非合法 UTF-8——可接受，由模型自行判断。
- `<codecvt>` 弃用警告已由 `core/CMakeLists` 全局抑制（PathCodec 已配 `add_compile_definitions(_SILENCE_CXX17_CODECVT_HEADER_DEPRECATION_WARNING)`）。
- `ContentCodec.h` 是头文件，需登记进 `core/CMakeLists` 的 `qharness_core` 源列表（VS 工程可见）。
