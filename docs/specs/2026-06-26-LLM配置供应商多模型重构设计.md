# LLM 配置重构（供应商 + 多模型）设计文档

- 日期：2026-06-26
- 状态：待评审
- 关联：`docs/specs/2026-06-26-设置系统设计.md`（被重构的前身）、`docs/plans/2026-06-26-设置系统实现计划.md`

## 1. 背景与目标

当前 LLM 配置为扁平的 `LlmProfile`（每个 profile = name + providerType + baseUrl + apiKey + model + temperature + maxTokens），一个 profile 只对应一个模型。实际问题：**同一个供应商（同一 baseUrl + apiKey）下通常有多个模型**（如智谱的 glm-4 / glm-4-flash，Anthropic 的多个 claude 版本），扁平结构导致每个模型重复填一遍 baseUrl/apiKey，冗余易错。

本次重构把 LLM 配置改为**两级**：供应商（baseUrl + apiKey + providerType）下挂多个模型。配置类与 `setting.json` 读写相应修正。

## 2. 范围

### 2.1 包含
- `LlmProfile` 拆为 `LlmProvider`（供应商级：name/providerType/baseUrl/apiKey）+ `LlmModel`（模型级：name/temperature/maxTokens）。
- `Settings`：`_profiles`→`_providers`，`_activeProfileName`→`_activeProviderName` + 新增 `_activeModelName`。
- 序列化（ADL）+ `findActiveProvider`/`findModel` helper 重写（沿用 I1 缺字段容错）。
- `SettingsDialog` LLM Tab 重构：左侧供应商列表 + 右侧供应商设置 + 模型表格（增删行、radio 选当前）。
- `EngineThread` 装配逻辑：按 activeProvider + activeModel 创建 Provider。
- 测试更新；CLAUDE.md 更新。

### 2.2 非目标（YAGNI）
- 旧 `setting.json`（profiles 格式）迁移——不迁移，新格式读不到 providers 视为空配置（用户重配）。
- 跨供应商的模型共享/继承。
- providerType 之外的 API 格式扩展。
- 模型表格的拖拽排序、复制等高级操作。

## 3. 关键决策汇总

| 维度 | 决策 | 理由 |
|---|---|---|
| providerType | 保留在供应商级（一种 API 格式） | 同一下游只走 OpenAI/Claude 一种格式 |
| 激活表达 | `activeProviderName` + `activeModelName` 双选 | EngineThread 据此定位 url/apiKey + model name + 该模型参数 |
| 模型表格列 | 模型名 + temperature + maxTokens + 当前(radio) | 每模型独立参数；radio 单选 = activeModelName |
| 供应商"当前" | 左侧列表单击即设为当前 + ✓标记（无单独按钮） | 与模型 radio 单选一致；KISS |
| 旧配置迁移 | 不迁移，重配 | 格式不兼容；此前为测试配置；KISS |
| 访问模式 | 沿用纯数据 + 值注入（非单例） | 不变 |

## 4. 数据模型（`core/schema/Settings.h`）

```cpp
enum class ProviderType { OpenAI, Claude };   // 不变（含 NLOHMANN_JSON_SERIALIZE_ENUM、providerTypeToString/FromString）

class QH_API LlmModel {
public:
    std::string _name;                        // 模型名，如 "glm-4"
    std::optional<double> _temperature;       // 该模型自己的参数；nullopt = 不下发
    std::optional<int>    _maxTokens;
};

class QH_API LlmProvider {
public:
    std::string _name;                        // 供应商名，唯一标识（如 "智谱"）
    ProviderType _providerType = ProviderType::OpenAI;
    std::string _baseUrl;
    std::string _apiKey;
    std::vector<LlmModel> _models;            // 该供应商下多个模型
};

class QH_API Settings {
public:
    std::vector<LlmProvider> _providers;       // 替代旧 _profiles
    std::string _activeProviderName;           // 替代旧 _activeProfileName
    std::string _activeModelName;              // 新增：在 _activeProviderName 下查找
    bool _enableThinking = false;              // 不变
    std::string _workDir;                      // 不变
    std::vector<std::string> _enabledTools;    // 不变
};

QH_API void to_json/from_json(LlmModel);
QH_API void to_json/from_json(LlmProvider);
QH_API void to_json/from_json(Settings);

// 按 _activeProviderName 查找供应商；未设置/未命中返回 nullptr
QH_API const LlmProvider* findActiveProvider(const Settings& s);
// 在 provider._models 中按 name 查找模型；未命中返回 nullptr
QH_API const LlmModel* findModel(const LlmProvider& provider, const std::string& modelName);
```

`class` + 首行 `public:`；成员下划线前缀；ADL 序列化置于 `qh::schema`。

## 5. 序列化（ADL，沿用 I1 缺字段容错）

- `LlmModel`：`{name, temperature?, maxTokens?}`，optional nullopt 时 omitempty。
- `LlmProvider`：`{name, providerType, baseUrl, apiKey, models}`，`vector<LlmModel>` 自动序列化。
- `Settings`：空值 omitempty（`_providers`/`_activeProviderName`/`_activeModelName`/`_enableThinking`/`_workDir`/`_enabledTools` 空/默认时不输出）。
- `from_json` 全部用 `j.value(key, 默认值)`（缺字段容错，沿用已验证的 I1 模式：单字段笔误不致全量回退）。

JSON 形态示例：
```json
{
  "providers": [
    {"name":"智谱","providerType":"OpenAI","baseUrl":"https://open.bigmodel.cn/api/paas/v4","apiKey":"sk-xxx",
     "models":[{"name":"glm-4","temperature":0.7,"maxTokens":4096},{"name":"glm-4-flash"}]}
  ],
  "activeProviderName":"智谱",
  "activeModelName":"glm-4",
  "enableThinking": false,
  "enabledTools": ["bash"]
}
```

## 6. EngineThread 装配（签名不变，逻辑改）

```cpp
const schema::LlmProvider* provider = schema::findActiveProvider(_settings);
const schema::LlmModel* model = provider ? schema::findModel(*provider, _settings._activeModelName) : nullptr;
const bool valid = provider && model
                   && !provider->_baseUrl.empty() && !provider->_apiKey.empty()
                   && !model->_name.empty();
if (valid) {
    // 按 provider->_providerType 创建 ProviderOpenAI/Claude(provider->_apiKey, provider->_baseUrl, model->_name)
    // setTemperature/setMaxTokens 取自 model->_temperature / model->_maxTokens
} else {
    // 回退 MockProvider/MockTwoStageProvider（按 kind）+ post Warn「未配置有效供应商/模型，回退 mock」
}
```

工具装配、Engine 选择（kind + `_enableThinking`）、workDir 回退逻辑不变。

## 7. SettingsDialog LLM Tab UI

```
┌─ LLM 配置 Tab ───────────────────────────────────────────────┐
│ ┌─────────────────┐  供应商设置                                │
│ │ 智谱      ✓当前 │  名称:  [_________]                        │
│ │ Anthropic       │  类型:  [OpenAI ▾]                         │
│ │ 本地Ollama      │  BaseURL:[_________]                       │
│ └─────────────────┘  APIKey: [********] (密码态)               │
│ [新增供应商][删除供应商]                                        │
│ (单击列表项即设为当前供应商)                                    │
│                                                                │
│ 模型（该供应商下）：           [添加模型][删除模型]             │
│ ┌────────────┬─────────┬──────────┬──────┐                    │
│ │ 模型名     │ temp    │ maxTokens│ 当前 │  ← QTableWidget    │
│ ├────────────┼─────────┼──────────┼──────┤                    │
│ │ glm-4      │ 0.7     │ 4096     │  ●   │  ← radio 单选      │
│ │ glm-4-flash│         │          │  ○   │                    │
│ └────────────┴─────────┴──────────┴──────┘                    │
│ (前 3 列可编辑；"当前"列 radio 单选写 activeModelName)          │
└────────────────────────────────────────────────────────────────┘
```

- 左 `QListWidget`（供应商列表）：**单击某项即设为当前供应商**（写 `_activeProviderName`）+ 右侧同步显示该供应商供编辑；当前激活项标「✓当前」；`新增供应商`/`删除供应商` 按钮（无单独"设为当前"按钮）。
- 右上表单：选中供应商的 名称/类型下拉(BaseURL)/APIKey(密码态)，编辑回写选中供应商。
- 右下 `QTableWidget`（模型表格）：4 列——模型名(可编辑)/temperature(可编辑)/maxTokens(可编辑)/当前(`QRadioButton` 单选，写 `_activeModelName`)；`添加模型`(末尾加空行)/`删除模型`(删选中行)。
- 切换左侧选中供应商 → 表单与模型表格同步显示该供应商。
- 重名校验：供应商名唯一；同一供应商内模型名唯一（沿用 M4 模式：重名 QMessageBox::warning + 回滚）。
- `_loading` 守卫防回写循环（沿用现有机制）。
- 引擎/工具/工作目录 Tab 不变。

## 8. 激活与迁移

- 激活：供应商「设为当前供应商」按钮写 `_activeProviderName`；模型表格「当前」列 radio 写 `_activeModelName`。EngineThread 据二者定位。
- 迁移：**不迁移**。旧 `setting.json`（`profiles` 格式）经新 `from_json` 读不到 `providers` → `_providers` 空 → 用户重配。load 不报错（旧文件视为空配置，`lastError` 空）。
