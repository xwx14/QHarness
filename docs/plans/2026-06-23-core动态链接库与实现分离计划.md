# core 动态链接库化与实现分离实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 `core` 由静态库改为动态链接库（`qharness_core.dll`），并将所有内联函数体（含派生骨架 TODO 占位）从头文件分离到对应 `.cpp`。

**Architecture:** 先在 STATIC 下做声明/实现分离（5 个派生类拆 `.cpp`，行为零变化）；再原子地 DLL 化——新建 `qh_export.h` 手写 `QH_API` 宏（`QH_BUILDING_DLL` 切换 dllexport/dllimport），所有公共类整类 `QH_API` 导出，CMake 改 SHARED 并把 DLL 与全部 exe 统一输出到 `build/out/<config>`（测试 exe 必须与 core DLL 同目录）。

**Tech Stack:** C++17、CMake（VS 多配置生成器）、MSVC `/utf-8`、手写 `__declspec(dllexport/dllimport)` 宏、项目自带 TestHarness。

## Global Constraints

（取自设计文档 `docs/specs/2026-06-23-core动态链接库与实现分离设计.md` 与项目 CLAUDE.md）

- C++17（`/utf-8`，MSVC + VS 生成器）。
- `core` 模块**不依赖 Qt**（仅 STL + vendored nlohmann/json）。
- include guard 用 `#ifndef/#define/#endif`，命名 `QH_<模块>_<文件>_H`；`qh_export.h` 为 `QH_EXPORT_H`。
- 命名空间：`qh::{schema,tool,provider,engine,robot,memory}`；内部头根相对 include（`core/` 为 include 根）。
- 所有新增 `.h` 列入 `core/CMakeLists.txt` 源列表（VS 工程可见）。
- 源码注释中文；变量/函数驼峰命名。
- 导出宏：手写 `QH_API`（编译 core 时 CMake 注入 `QH_BUILDING_DLL` → `dllexport`，否则 `dllimport`）；导出粒度=整类 `class QH_API X`。
- CRT：core/app/tests 均用 MSVC 默认动态 CRT（`/MD`/`/MDd`），保持一致。
- git 提交信息中文；提交前需用户确认（项目约定：未经要求不执行 git 操作）。
